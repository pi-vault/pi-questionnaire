## When to use it

Use `questionnaire` when an agent should pause and gather structured input before continuing, for example:

- clarifying requirements before implementation
- collecting user preferences or tradeoffs
- confirming scope, priorities, or rollout choices
- batching several related decisions into one interaction

Use normal conversation instead when the user is already giving free-form feedback and there is no structured decision to capture.

## What the user sees

The tool opens an interactive terminal UI with one tab per question and a final review screen.

Current interaction paths:

- **Single-choice** — select one option
- **Multi-choice** — select one or more options
- **Type something.** — available on single-choice questions by default for custom text
- **Chat about this** — available by default to signal that the question should be discussed instead of answered directly
- **Notes** — optional per-question notes for extra context without changing the main answer

When the user submits, the tool returns a compact text summary for the agent plus structured result details.

## Question format

`questionnaire` supports two question types.

### Single-choice

```ts
{
  type: "single-choice",
  id: "scope",
  header: "Scope",
  prompt: "Which scope should we ship first?",
  options: [
    { value: "small", label: "Small", description: "Minimal release scope" },
    { value: "full", label: "Full", description: "Broader first release" }
  ],
  recommendation: "small",
  allowOther: true,
  allowChat: true
}
```

### Multi-choice

```ts
{
  type: "multi-choice",
  id: "features",
  header: "Features",
  prompt: "Which features do you want in the first release?",
  options: [
    { value: "notes", label: "Notes" },
    { value: "chat", label: "Chat handoff" },
    { value: "review", label: "Review screen" }
  ],
  recommendation: ["notes", "review"],
  allowChat: true
}
```

## Example tool call

```ts
questionnaire({
  questions: [
    {
      type: "single-choice",
      id: "scope",
      header: "Scope",
      prompt: "Which scope should we ship first?",
      options: [
        {
          value: "small",
          label: "Small",
          description: "Minimal release scope"
        },
        {
          value: "full",
          label: "Full",
          description: "Broader first release"
        }
      ],
      recommendation: "small",
      allowOther: true,
      allowChat: true
    },
    {
      type: "multi-choice",
      id: "checks",
      header: "Checks",
      prompt: "Which release checks should we run?",
      options: [
        { value: "lint", label: "Lint" },
        { value: "types", label: "Typecheck" },
        { value: "tests", label: "Tests" }
      ],
      recommendation: ["lint", "types", "tests"],
      allowChat: true
    }
  ]
})
```

## Example result summary

A submitted questionnaire returns a text summary in the style below:

```text
Scope: user selected: 1. Small
Scope note: "keep the first release tight"
Checks: user selected: 1. Lint, 2. Typecheck, 3. Tests
```

Other possible result lines:

```text
Scope: user wrote: "Ship only the core questionnaire flow"
Features: user wants to discuss this question
```

## Behavior notes

- Interactive TUI mode is required. In non-interactive mode the tool returns an error.
- Cancelled questionnaires return a cancelled result instead of partial submission.
- `allowOther` applies to single-choice questions only.
- `allowChat` can be enabled for both single-choice and multi-choice questions.
- Keep questionnaires small when possible. A few well-chosen questions work better than a long survey.
